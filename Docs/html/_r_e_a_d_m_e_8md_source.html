6 Poppy 2.0 communication lib
7 ===========================
8 
9 The role of this code is to manage the Poppy 2.0 communication stack between all modules.
10 
11 You can use it to develop you own Poppy 2.0 module code.
12 
13 To understand how this protocole work, please read the [protocol definition](doc/protocol_definition.md).
14 
15 How to start a new Poppy 2.0 module code project
16 ------------------------------------------------
17 
18 First of all you have to fork this repository to create your hown module on your github account.
19 To avoid any trouble during future updates of the poppy-pross lib, take care to not modifying any files in the src folder or create a specific branch for all your developpements.
20 __________________________________________________________________________
21 
22 In this repo travis is used to create documentation, update readme, and pass some tests.
23 To activate all this things you have to follow some steps :
24  - enable your repo on [travis](https://travis-ci.org/) and on [coveralls](https://coveralls.io).
25  - [Create an encrypted token for travis.](doc/travis_encrypt.md)
26  - replace your encrypted token to the .travis.yml file of this repo.
27 
28 __________________________________________________________________________
29 
30 To have a perfect integration and declaration of your module into a poppy 2.0 network you have to:
31  - specify your module in the mod_list.h list.
32  - specify your MCU on the Makefile.
33  - specify your MAINCLOCK frequency on the Makefile.
34  - specify your SCLFREQ to specify the I2C max speed on the Makefile.
35  - create your specific hal folder if you need. To do that please read the [new hal creation documentation](doc/hal_creation.md).
36 
37 You can use the [template.c](template.c) file to format your main file and use it as base.
