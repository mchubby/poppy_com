00006 Poppy 2.0 communication lib
00007 ===========================
00008 
00009 The role of this code is to manage the Poppy 2.0 communication stack between all modules.
00010 
00011 You can use it to develop you own Poppy 2.0 module code.
00012 
00013 To understand how this protocole work, please read the [protocol definition](doc/protocol_definition.md).
00014 
00015 How to start a new Poppy 2.0 module code project
00016 ------------------------------------------------
00017 
00018 __________________________________________________________________________
00019 
00020 First of all you have to fork this repository to create your hown module on your github account.
00021 To avoid any trouble during future updates of the poppy-pross lib, take care to not modifying any files in the poppy-com folder or create a specific branch for all your developpements.
00022 __________________________________________________________________________
00023 
00024 In this repo travis is used to create documentation, update readme, and pass some tests.
00025 To activate all this things you have to follow some steps :
00026  - enable your repo on [travis](https://travis-ci.org/) and on [coveralls](https://coveralls.io).
00027  - [Create an encrypted token for travis.](doc/travis_encrypt.md)
00028  - replace your encrypted token to the .travis.yml file of this repo.
00029 
00030 __________________________________________________________________________
00031 
00032 To have a perfect integration and declaration of your module into a poppy 2.0 network you have to:
00033  - specify your module in the mod_list.h list.
00034  - specify your MCU on the Makefile.
00035  - specify your MAINCLOCK frequency on the Makefile.
00036  - specify your SCLFREQ to specify the I2C max speed on the Makefile.
00037  - create your specific hal folder if you need. To do that please read the [new hal creation documentation](doc/hal_creation.md).
