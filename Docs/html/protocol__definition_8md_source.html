4 Hardware and data flow
5 ----------------------
6 
7 Poppy communication protocol use 3 data wire :
8 
9  - SCL
10  - SDA
11  - PTP
12 
13 Wire SCL and SDA are used to manage high-speed **I2C** (3.6Mhz). This bus is the **main communication way**. The **PTP** wire means "point to point", he manage hardware detection by passing tokens and allow to comminucate directly with the next or previous module with high speed serial communication.
14 
15 All messages I2C or PTP have the same structure :
16 
17  - Address
18  - msg_type
19  - msg_size*
20  - msg[0]*
21  - ...*
22  - msg[msg_size-1]*
23  - CRC*
24 
25 \* this part of the message can be missing on somes network level messages, but you will probably don't have to seal with it!
26 
27 Protocol levels
28 ---------------
29 
30 The Poppy communication stack have different levels of messages :
31 
32  - **user messages** (accessible by the end user)
33  - **module messages** (specific to a module type)
34  - **network messages** (network management messages)
35 
36 Each of these level have his hown **msg_type field**. **Network messages** are prioritary on **module messages** and **module messages** are prioritaty on **user messages**.
37 
38 If you need to use some modules and write your hown code for your own robot you only have to deal with **user messages**.
39 If you are a module creator you will need to deal with module level messages.
40 You probably don't have to deal with **network messages**, but if you have any question please describe it in the [Poppy project forum](https://forum.poppy-project.org).
