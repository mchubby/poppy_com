4 The Poppy communication stack have two levels of messages.

7 The module level messages
8 -------------------------
9 
10 You only have to deal with this kind of message in your module application.
11 These messages are dedicated to your module application, and you have to define it.
12 
13 The single limitation is to match with the msg_t struct.
14 
15 All messages send or received should have a register and a
16 size.
17 A typical message on the bus will be :
18  - Slave addr
19  - Register
20  - Size
21  - Data[0]
22  - ...
23  - Data[size - 1]
24 
25 If you have a message with only a register needed you have to set size to 0.
26 The message transmitted will be :
27 - Slave addr
28 - Register
29 - 0
30 
31 Your registers definition for your application should be an enum who start at "last_reg_val" value.
